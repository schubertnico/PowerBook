I. Über PowerBook
[ Top ]

PowerBook ist ein klassisches Gästebuch-System auf PHP- und MySQL-Basis, ursprünglich im Jahr 2002 von Axel "Expandable" Habermaier entwickelt. 2025 wurde das Script von Nico Schubert / SchubertMedia vollständig auf PHP 8.4 portiert und im April 2026 einem ausführlichen Sicherheits-Audit unterzogen (alle 14 gefundenen Bugs wurden behoben, dazu drei priorisierte Workflow-Verbesserungen umgesetzt).

Das Projekt ist in aktiver Pflege:
  • Projekt-Webseite: https://www.powerscripts.org
  • Projektbereich: https://www.powerscripts.org/projects-5.html
  • Repository: github.com/schubertnico/PowerBook
Features im Überblick
  • Gästebuch-Frontend für Besucher: Eintrag schreiben, lesen, suchen, paginieren
  • AdminCenter mit Rollen- & Rechtemanagement
  • BBCode-Textformatierung ([b], [i], [u], [small]) + Auto-Linking
  • Smilies/Emoticons, umschaltbar pro Eintrag
  • Icons für Einträge (Frage, Ausruf, Smileys)
  • Anti-Spam mit IP-Zeitsperre (konfigurierbar)
  • Token-basierter Passwort-Reset mit 30-Min-TTL
  • CSRF-Schutz + Session-Fixation-Schutz
  • SMTP-Versand für Benachrichtigungen (Mailpit im Dev-Setup)
II. Schnellstartanleitung
[ Top ]

Docker — in 60 Sekunden lauffähig: 
# 1. Repository klonen
git clone https://github.com/schubertnico/PowerBook.git
cd PowerBook/.docker

# 2. Container starten (Apache+PHP, MySQL, Mailpit)
docker compose up -d --build

# 3. Installation im Browser ausführen
#    -> http://localhost:8080/install_deu.php
#    Auf "Installation starten" klicken.
#    Das angezeigte Initial-Passwort SOFORT notieren!

# 4. Loslegen
# Frontend (Gästebuch):  http://localhost:8080/pbook.php
# Admin-Center:           http://localhost:8080/pb_inc/admincenter/
# Mailpit (Test-Mails):   http://localhost:8031
Hinweis zum Initial-Passwort (Stand April 2026): Bei der Installation wird ein zufälliges 16-stelliges Hex-Passwort erzeugt (kein hardcoded powerbook mehr). Das Passwort erscheint einmalig auf der Erfolgsseite. Bitte sofort notieren oder direkt im AdminCenter ändern.

IV. Installation (ausführlich)
[ Top ]

A) Mit Docker (empfohlen für Entwicklung)

Das Repository enthält unter .docker/ ein vollständiges Compose-Setup:
ServiceImageHost-Port
webApache 2.4 + PHP 8.48080
dbMySQL 8.03314
mailAxllent MailpitSMTP 1031, Web-UI 8031
git clone https://github.com/schubertnico/PowerBook.git
cd PowerBook/.docker
docker compose up -d --build
Anschließend http://localhost:8080/install_deu.php aufrufen und auf "Installation starten" klicken. Die Installation:
  1. legt die MySQL-Tabellen pb_admins, pb_config, pb_entries an (mit Spalten reset_token + reset_token_expires für den sicheren Recovery-Flow),
  2. erzeugt einen Standard-Admin PowerBook mit zufälligem 16-Hex-Passwort,
  3. zeigt dieses Passwort einmalig in einer Hervorhebungs-Box,
  4. erstellt das Lock-File .installed, sodass install_deu.php nicht versehentlich erneut ausgeführt werden kann (HTTP 403).

B) Klassisches Hosting (LAMP)

  1. Dateien per FTP/SCP auf den Webserver übertragen.
  2. pb_inc/mysql.inc.php anpassen: 
    $config_sql_server   = 'localhost';
$config_sql_user     = 'powerbook';
$config_sql_password = 'IhrSicheresPasswort';
$config_sql_database = 'powerbook';
  3. https://ihre-domain.tld/install_deu.php im Browser aufrufen.
  4. Initial-Passwort notieren.
  5. Empfehlung: install_deu.php zusätzlich per .htaccess schützen. Das .installed-Lock greift automatisch bei Re-Install-Versuchen.
V. AdminCenter
[ Top ]

URL: http://<host>/pb_inc/admincenter/
BereichZweck
HomeDashboard mit System-Info und Schnellnavigation
Einträge verwaltenListe, Bearbeiten, Löschen, Statement schreiben
Einträge freischaltenSichtbarkeit (R/U) — abhängig von Konfiguration
Admins verwaltenHinzufügen, Bearbeiten, Löschen, Rechte (Konfig/Admins/Einträge/Release)
KonfigurationSpam-Check, Anzeige, Smilies/BBCode/ICQ, E-Mail, Eintrags-Design
LizenzMIT-Lizenztext
LogoutBeendet die Session

Passwort vergessen?
  • Im AdminCenter ?page=password aufrufen.
  • Admin-Name oder E-Mail eingeben → Reset-Link per Mail (30 Min gültig).
  • Erst durch Klick auf den Link und Setzen des neuen Passworts wird das alte ungültig.
VI. Sicherheit & Audit 2026
[ Top ]

Im April 2026 wurde ein vollständiges Sicherheits-Audit des Userbereichs durchgeführt. Alle 14 dokumentierten Bugs sind behoben und drei priorisierte Workflow-Verbesserungen umgesetzt.

KategorieMaßnahme
SQL-InjectionPDO Prepared Statements für alle Queries
XSSKonsequentes Output-Escaping; keine doppelten Escapes mehr im Preview
CSRFToken in jedem Formular, Rotation nach erfolgreichen Aktionen
Session-Fixationsession_regenerate_id(true) nach jedem Login
Passwort-Speicherungpassword_hash(PASSWORD_DEFAULT), Auto-Migration
Passwort-ResetToken-Flow mit 30-Min-TTL, kein Sofort-Reset
User-EnumerationGenerische Recovery-Response
LFI / Path-TraversalWhitelist $allowedPages im AdminCenter
Direktaufruf-SchutzPB_ENTRY-Konstanten-Guard
Installation-Schutz.installed-Lock verhindert Re-Install (HTTP 403)
Standard-PasswortZufällig, bin2hex(random_bytes(8))
Admin-Add-FallbackBei SMTP-Fehler wird Initial-Passwort im UI gezeigt
Längen-ValidierungServerseitig für Name (100), Text (5000), E-Mail (250), URL (255)
Cookie-FlagsHttpOnly + SameSite=Strict
E-Mail-HeaderSanitisierung aller Recipient-Header
